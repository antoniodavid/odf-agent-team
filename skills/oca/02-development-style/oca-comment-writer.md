## When to Use

Load this skill whenever you write a comment that another human will read: PR review feedback, issue replies, maintainer responses, or async project updates.

## Hard Rules

| Rule | Requirement |
|------|-------------|
| Be useful fast | Start with the actionable point. Do not recap the whole PR before feedback. |
| Be warm and direct | Sound like a thoughtful teammate, not a corporate bot. |
| Keep it short | 1-3 short paragraphs or a tight bullet list. |
| Explain why | Give the technical reason when asking for a change. |
| Avoid pile-ons | Comment on the highest-value issue, not every tiny preference. |
| No AI attribution | No Co-Authored-By trailers or "Generated by" lines. |

## Decision Gates

| Situation | Formula |
|-----------|---------|
| Request change | Direct observation → Why it matters → Concrete next action |
| Approve with note | Positive assessment → Optional improvement suggestion |
| Ask clarification | What's unclear → What I expected → Question |
| Follow-up | What was done → What remains → Suggestion |

## Output Contract

Return comment text ready to paste. Format: Direct observation or request, why it matters (only if needed), concrete next action.
